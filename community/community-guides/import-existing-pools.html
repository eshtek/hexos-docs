Who this guide is for

This guide will help you: 

  • Migrate from TrueNAS to HexOS
  • Move HexOS to new hardware when a reinstall is required
  • Reinstall HexOS on the same hardware

specially when you do not want to wipe your storage pools as part of the process.

Best practice would be a fresh install of HexOS and with new storage pools, with data added to the new storage pools from backup afterwards. However, if you are in a hurry or perhaps have:

  • a large amount of data in your storage pools
  • a large number of drives in the storage pools
  • complex app configurations

then this guide will show you how to keep your existing storage pools and import them to a new HexOS install.

The guide does not assume significant prior knowledge of the software involved, and all steps are described thoroughly. If at any point you do not feel confident, please pause and ask for help - this is a high risk process as a mistake could make your data inaccessible.

Warning

You must have backups of your data before you attempt this process: without adequate backups you can lose your data during this process.

  1. Have backups of everything
  2. Verify they still work
  3. Try to copy a small set of files to prove out this isn’t a new problem that you’re not aware of

Exporting TrueNAS configuration and keys

In this section of the guide we will complete the following tasks:

  • Log in to TrueNAS
  • Turn off 2FA
  • Download TrueNAS config file and back it up
  • Download any encryption keys and back them up
  • Find our boot pool drive and storage pool drives serial numbers
  • Check we are ready to install HexOS

By exporting the TrueNAS configuration you are backing up

Log into TrueNAS Web Interface

From HexOS

If you are starting in HexOS, go to https://deck.hexos.com/ > Settings > TrueNAS

This will open a new window in your browser. You may need to click through a security warning about a self signed SSL certificate, for example, in Firefox I clicked Advanced, and Accept the risk and continue. 

Next you will see the TrueNAS login. 

The username is truenas_admin and the password is the one chosen when you installed HexOS.

From TrueNAS

If you are not yet using HexOS, enter TrueNAS by going to the login screen at your server IP address. For example https://192.168.1.142/ui/signin.

Your username as password were set during TrueNAS installation.

Turn off 2FA

If 2FA is enabled you will not be able to claim your server when installing HexOS. Check that 2FA is disabled. Go to System > Advanced Settings then scroll down to Global Two Factor Authentication and check that 2FA is showing as disabled.

 If it is showing as enabled, click Configure, un-check the Enable Two Factor Authentication Globally box and click Save

Download TrueNAS Configuration 

Go to System > Advanced Settings > Manage Configuration > Download File

You may need to scroll down to see the System menu item

The Manage configuration dropdown is on the top right of the screen.

Check that the file has downloaded. Make a note of the TrueNAS version number included in the file name (in this case 25.10.4) - you will need this later.

Back up this file to a secure local place, such as a usb drive.

Download Keys for Encrypted Datasets

If you have data encrypted with a key in your storage pool, the keys must be downloaded. If this step is skipped you will lose your data.

To see if you have encrypted data go to Datasets from the main menu in TrueNAS. In the Encryption column, datasets that are encrypted are marked as Locked or Unlocked. The rest are marked Unencrypted.

To find out if a dataset is key encrypted, click on the entry then scroll down to Encryption. If there is a button for Export Key, then the data is encrypted with a key. You need this key be able to access the data again later: without the key the data is unreadable. 

Click the Export Key button. You will be reminded that each encrypted dataset has it's own key, and you must download each key separately.

When you have downloaded they key, check that it looks similar to the example key below:

Then back up the key to a safe local place, such as a USB drive.

If there is no Export Key button showing, this means the data is protected with a passphrase and not a key and so this backup step is not needed.

To confirm this click Edit (you may need to unlock first) and check that the Encryption Type* dropdown is showing Passphrase. Encrypted folders set up in HexOS always use a passphrase.. 

f you have any doubts or questions, please contact HexOS support, this step is critical to get right.

Find the serial numbers of your boot dive and storage pool drives

To prevent data loss, we want to make sure we know which drives are pool drives and which drives are boot drives in the machine. We're going to unplug the pool drives now to make sure they don't get overwritten.

First, in TrueNAS go to Storage > Disks

You will see a list of all the drives in the machine. Note that the name of the boot drive is boot-pool. The storage pool drives, which have our data on, will have the name given when the storage pool was created, in this case “HDDs".

Copy down the drive serial numbers and their pool name/use. You can click an entry to see more of the serial number if needed. 

When we boot down the server, all the storage pool drives will be unplugged.

Final Checks

You should now have a

  • Copy of the config file any encryption keys
  • Backup of these
  • Fully backup of the data on your pools which you have verified is working
  • Note of your TrueNAS version number
  • Sheet of paper with the drive serial numbers on

Preparing to Install HexOS

In this section we will:

  • Shut down the server
  • Unplug the storage pool drives
  • Make any other hardware updates

Shut down the server

First, shut down the server. From TrueNAS this can be done by clicking the power button icon and Shut Down

then check confirm  and click Shut Down

When the browser starts to show a connection error, the server has shut down.

In HexOS click the power button then Shut Down. 

Then confirm by clicking Yes, shutdown now

When the browser starts to show a connection error, the server has shut down.

Unplug your storage pool drives

At this stage, unplug all the drives that make up the storage pools. It is essential to do this, to prevent accidental data loss during HexOS install.

These drives can be identified by the serial number (or model number) that was noted down earlier.

Make any hardware changes

Make any other planned hardware changes to your system now also. 

In this case, the same boot drive is used, so this remains plugged in. The serial number matches the boot-pool drive in TrueNAS.

Install HexOS

Follow the HexOS installation guide here.

When you get to the step Select Installation Drive there should only be one drive listed.  Check that this matches the description of the drive you want to install HexOS on.

At the end of the install process, you will see the following Console setup screen. 

Do not continue following the docs to complete the HexOS complete setup. First, log into TueNAS by going to the IP address shown on the Console setup screen.

Log in to TrueNAS.

Upgrade TrueNAS

The next step is to update TrueNAS to the same version as our previous instance. You can find this from the config file name, or in your notes from earlier steps. [notes on updating truenas while using hexos genrally?)[[TODO]

After loging in, TrueNAS should open on the dashboard. 

Click the Updates Available button.

 In this instance there is only one update to make our TrueNAS version numbers match. If the options available do not allow you to update to the correct version, please contact support [TODO] or something.

Click Install Update.

Download the config and settings and password as suggested by clicking Save Configuration.

Then click Install.

The system will run the update and reboot

When the reboot has finished and the Console Setup screen is showing on the monitor again, the next step is to upload the old config.

Plug drives back in and reboot

 

Import your pool config

  • Go to Storage > Import Pool > find your zpool
  • Got an error? Check step 9 again. Make sure you are using the same version of TrueNAS as your old boot drive. 
  • Go into System > General Settings > Manage Configuration > Upload File
  • After reboot, go to Apps > Configuration > Unset Pool
  • Restart TrueNAS
  • Go back into the web interface, go to Apps > Configuration > Choose Pool
  • Your apps should show back up and you should be able to start them.
  • Go to the HexOS web interface. 
  • If you haven't already, unclaim your old server and claim the new one
  • The hardware check screen should give you a warning about an existing pool. This is good, go onto the next screen and skip creating pool. 
  • Give your server a name and you should be good to go!
