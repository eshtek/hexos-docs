Method 1: The Automated .reg File (Fastest for Customers)

This method allows customers to double-click a single file to apply the fix safely.

Instructions for the Customer:

Open Notepad on your computer.

Copy and paste the exact text block below into the blank document:

Plaintext

 

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters]
"RequireSecuritySignature"=dword:00000000

Click File > Save As...

Change the "Save as type" dropdown to All Files (.).

Name the file Fix-TrueNAS-Sharing.reg (the .reg extension is critical).

Save it to your Desktop, then double-click it.

Accept the Windows UAC security prompts to apply the change, then reboot your PC.

Method 3: The Graphical UI (Group Policy)

For enterprise customers or those running Windows 11 Pro/Enterprise who prefer using standard administrative tools:

Press Win + R, type gpedit.msc, and hit Enter to open the Local Group Policy Editor.

Navigate down the tree layout:

Computer Configuration $\rightarrow$ Windows Settings $\rightarrow$ Security Settings $\rightarrow$ Local Policies $\rightarrow$ Security Options

Locate the policy named:

Microsoft network client: Digitally sign communications (always)

Double-click it, set it to Disabled, click Apply, and reboot.
